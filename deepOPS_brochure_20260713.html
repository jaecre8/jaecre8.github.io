동작 방식

6단계 자동 작업 플로우

지금 손으로 하고 있는 작업 순서를 그대로 따라가되, 터미널 대신 채팅으로 진행합니다.

완전 자동화

처음부터 끝까지 AI가 모든 단계를 자동 수행

시간 절약

반복 작업을 줄여 업무 시간을 절감

안전한 처리

중요 작업은 사용자 승인 후 최종 반영

  1. 01 STEP

    티켓 번호 입력

    채팅창에 JIRA 티켓 번호만 입력하면 시작됩니다. 알려줄 내용이나 첨부파일이 더 있다면 함께 전달할 수 있어요.

    입력 예시

    ITO-1234 + 추가 정보

  2. 02 STEP

    분석 리포트 생성

    이슈 내용, 댓글, 첨부파일, 관련 코드를 훑어서 영향 범위·수정 전략·테스트케이스가 담긴 리포트를 자동으로 만듭니다.

    자동 분석 항목
    영향 범위 수정 전략 테스트케이스
  3. 03 STEP

    레포 선택

    이슈 키워드를 바탕으로 관련 있을 법한 레포를 추천하지만, 최종 선택은 항상 사람이 채팅 버튼으로 직접 확정합니다.

    사용자 확인

    추천 레포 리스트 제공

  4. 04 STEP

    코드 수정 시도

    선택한 레포에서 실제 코드 수정을 시도합니다. 파일을 읽고 고치는 권한만 있고, 터미널 명령을 실행하는 권한은 없습니다.

    AI 작업 범위

    읽기 / 수정 권한만 보유

  5. 05 STEP

    Diff 검토

    무엇이 바뀌었는지 diff로 정리해서 보여줍니다. 진행되는 동안의 로그도 채팅창에서 그대로 확인할 수 있어요.

    검토 포인트

    변경 내용 시각화 + 설명

  6. 06 STEP

    결정

    push · keep · discard · abandon 중 하나를 버튼으로 고르면 끝. push를 선택했을 때만 GitLab에 브랜치가 올라가고 Draft MR이 만들어집니다.

    최종 선택

    사용자 승인 후 반영

모든 단계에서 사용자의 확인과 승인을 거쳐 안전하게 진행됩니다.

결과물

대화가 끝나면, 파일로 남습니다

영향 범위·수정 전략·테스트케이스가 정리된 리포트가 자동으로 저장됩니다. 이전에 처리한 티켓들도 옆 패널에서 언제든 다시 열어볼 수 있어요.

AI는 제안하고, 결정은 사람이 합니다

코드를 직접 고친다는 말이 불안하게 들릴 수 있어서, 처음부터 이렇게 선을 그어뒀습니다.

Bash 권한 없음

코드 실행 보호
ON

코드 수정 권한은 파일 읽기·쓰기까지만 허용됩니다. 터미널 명령을 실행할 권한은 처음부터 주지 않습니다.

  • 파일 시스템 접근 제한
  • 명령 실행 차단
  • 읽기·쓰기만 허용

자동 push 없음

변경 사항 통제
ON

어떤 경우에도 사람이 diff를 확인하고 결정 버튼을 누르기 전까지 GitLab에는 아무 것도 올라가지 않습니다.

  • 자동 commit 차단
  • 자동 push 차단
  • 사람의 승인 후 반영

언제든 중지

즉시 제어 가능
ON

진행 중인 작업은 상단 중지 버튼으로 즉시 멈출 수 있고, 원하는 시점에 새 티켓으로 다시 시작할 수 있습니다.

  • 즉시 중지 가능
  • 작업 상태 보존
  • 새 티켓으로 재시작

추천은 참고용

사람의 최종 판단
ON

레포 추천과 분석 리포트는 모두 참고 자료입니다. 실제 판단과 최종 선택은 항상 운영자의 몫입니다.

  • 추천은 참고 자료
  • 최종 판단은 사람
  • 운영자의 선택 존중
